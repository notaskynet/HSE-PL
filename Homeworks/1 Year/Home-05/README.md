## Основы работы с CMake и googletest

### Ответвление
Необходимо создать ответвление (fork) моего репозитория Home-05 (название
оставить без изменения, а видимость поставить приватной). Назничить меня в
новом репозитории соавтором с правами администратора. Вики, Задачи а также
Проекты отключить.

В локальной копии создавать ветки (branch) по необходимости.

### Этап разработки
Разработать абстрактный класс StreamProcessor содержащий два закрытых поля
типов std::istream и std::ostream соответственно, открытый конструктор
общего вида позволяющий инициализировать эти поля, а также чисто виртуальный
метод Process.

Разработать шаблон класса RLE, унаследованный от класса StreamProcessor, 
выполняющий прямое и обратное преобразование Run-Length-Encoding для входного 
потока (результат записывается в выходной поток). Параметром шаблона является 
число бит, используемых для для записи числа повторений (по умолчанию 8). 
Применить схему, при которой:
  * если старший бит сброшен, то число повторений занимет число бит, согласно 
    значению параметра;
  * если старший бит установлен, то за ним следует ровно 1 бит со значением 1, 
    а число повторений равно 1.

С помощью системы CMake обеспечить возможность сборки программы для работы,
а также для проведения тестов с использованием библиотек googletest/gmock и
формирования отчета о покрытии.

Для всех разработанных функциональных блоков подготовить тесты:
  * с использованием библиотеки googletest для функций;
  * с использованием библиотеки gmock для классов и их методов.
Обеспечить 100% покрытие тестами.
